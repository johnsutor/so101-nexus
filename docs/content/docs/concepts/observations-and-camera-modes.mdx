Every SO101-Nexus environment returns observations as either a flat state vector or a dictionary containing state and camera data. The content of the state vector depends on the **observation mode**.

## Observation Modes

The `obs_mode` config parameter controls what information is included in the `"state"` key of the observation. There are two modes:

### `obs_mode="state"` (default)

The state vector contains full privileged simulator information, including TCP pose, object positions, grasp flags, and relative vectors. This is useful for state-based reinforcement learning where the policy has access to ground-truth information.

| Task | Dimensions | Key Components |
| --- | --- | --- |
| PickLift | 18 | TCP pose, grasp flag, object pose, TCP-to-object vector |
| PickAndPlace | 24 | TCP pose, grasp flag, target position, object pose, TCP-to-object and object-to-target vectors |
| Reach | 10 | TCP pose, TCP-to-target vector |
| LookAt | 10 | TCP pose, gaze-to-target vector |
| Move | 10 | TCP pose, TCP-to-target vector |

Reach, LookAt, and Move share the same 10-dimensional layout. The semantic difference is in the third component: Reach and Move measure TCP-to-target distance, while LookAt measures gaze-to-target orientation error.

The exact per-index observation breakdown is documented on each [environment's reference page](/docs/environments).

### `obs_mode="visual"`

The state vector contains only the 6 joint positions, matching what a real robot's joint encoders would report. This mode is designed for vision-based policies that should learn to locate objects using camera images rather than ground-truth state. It requires `camera_mode="wrist"`.

| All Tasks | Dimensions | Key Components |
| --- | --- | --- |
| Any | 6 | Joint positions (shoulder_pan, shoulder_lift, elbow_flex, wrist_flex, wrist_roll, gripper) |

The full privileged state vector is still available in the `info` dict under the `"privileged_state"` key for logging or debugging purposes.

```python
from so101_nexus_core import PickConfig
from so101_nexus_mujoco.pick_env import PickLiftEnv

cfg = PickConfig(camera_mode="wrist", obs_mode="visual")
env = PickLiftEnv(config=cfg, camera_mode="wrist")
obs, info = env.reset()

print(obs["state"].shape)              # (6,) — joint positions only
print(obs["wrist_camera"].shape)       # (224, 224, 3) — camera image
print(info["privileged_state"].shape)  # (18,) — full state for debugging
env.close()
```

## Inspecting Observations

```python
import gymnasium as gym
import so101_nexus_mujoco

env = gym.make("MuJoCoPickLift-v1")
obs, info = env.reset()
print(f"Observation shape: {obs.shape}")  # (18,)
print(f"TCP position: {obs[:3]}")
print(f"Is grasped: {obs[7]}")
env.close()
```

## Camera Modes

By default, environments return state-only observations. To include camera images, set the camera mode when creating the environment.

On **MuJoCo**, pass `camera_mode` as a keyword argument to `gym.make()`. The accepted values are `"state_only"` (default, no camera) and `"wrist"` (wrist-mounted camera). When `"wrist"` is used, the observation becomes a dictionary:

```python
import gymnasium as gym
import so101_nexus_mujoco

env = gym.make("MuJoCoPickLift-v1", camera_mode="wrist")
obs, info = env.reset()
# obs["state"] -> flat state vector
# obs["wrist_camera"] -> image array
```

On **ManiSkill**, camera rendering is configured through the environment config's `camera_mode` field. Use `obs_mode="state"` for state-only observations.

Camera resolution and wrist camera field-of-view can be configured through `CameraConfig`. See the [Configs API reference](/docs/api/configs) for all parameters.

## Choosing the Right Mode

| Use Case | `obs_mode` | `camera_mode` | Why |
| --- | --- | --- | --- |
| State-based RL training | `"state"` | `"state_only"` | Policy uses privileged state directly |
| Vision-based RL training | `"visual"` | `"wrist"` | Policy learns from camera images; prevents cheating via ground-truth state |
| Teleop dataset recording | `"visual"` | `"wrist"` | Matches real robot sensor setup (joints + camera) |
| Debugging / visualization | `"state"` | `"wrist"` | Full state + camera for analysis |
