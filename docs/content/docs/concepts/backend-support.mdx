SO101-Nexus uses MuJoCo as its default simulation backend. A MuJoCo Warp backend adds native GPU-parallel batching for reinforcement learning, and covers all five tasks (Reach, LookAt, Move, PickLift, PickAndPlace).

## Task Availability

All five tasks run on the SO-101 arm:

| Task | Robot |
|------|-------|
| PickLift | SO-101 |
| PickAndPlace | SO-101 |
| Reach | SO-101 |
| LookAt | SO-101 |
| Move | SO-101 |

## Vectorization

MuJoCo environments are vectorized through Gymnasium's standard `gym.vector.SyncVectorEnv` wrapper, which runs multiple environment instances on CPU.

The MuJoCo Warp backend instead batches simulation natively on the GPU. Construct any task with `gym.make_vec("WarpPickLift-v1", num_envs=N, device="cuda", vectorization_mode="vector_entry_point")` (env IDs `WarpReach-v1`, `WarpLookAt-v1`, `WarpMove-v1`, `WarpPickLift-v1`, `WarpPickAndPlace-v1`); reset and step return torch CUDA tensors of shape `(num_envs, ...)`. The Warp envs route the same composable state observation components as MuJoCo (joint positions, end-effector pose, grasp state, object and target offsets, gaze direction); camera observations are not supported. Because all worlds share one model, two behaviors diverge from CPU MuJoCo by design: per-episode cube and target colors are fixed to the first configured color (the model's `geom_rgba` is global), and `WarpPickLift` supports a single `CubeObject` target (no distractors, YCB, or mesh objects). The physics also diverges (implicit integrator, no noslip constraint), so policies may need re-tuning when moving between backends. Install the backend with the `warp` extra (`pip install "so101-nexus[warp]"`, which needs an NVIDIA GPU and CUDA >= 12.4).

## Objects

MuJoCo supports `CubeObject`, `YCBObject`, and `MeshObject`. MuJoCo environments use the same composable observation components (`observations` parameter). See [Observations](/docs/concepts/observations-and-camera-modes) for details.

For details on object types, see the [Objects API reference](/docs/api/objects). For observation configuration, see [Observations](/docs/concepts/observations-and-camera-modes).
